# ELYSIUM — UI/UX Architecture & Development Guide > A modern educational platform with minimalist, high-contrast aesthetics reminiscent of Notion, Linear, and contemporary productivity tools. --- ## 1. Design System & Token Layer ### 1.1 CSS Variables (`:root` Block) All styling derives from predefined tokens. **Never hardcode hex colors or dimensions.** ```css :root { /* Color Palette - Monochrome & Muted */ --white: #ffffff; /* Card, sidebar backgrounds */ --off: #f7f7f5; /* App background, hover states */ --light: #efefed; /* Subtle hover states */ --mid: #c8c8c4; /* Section labels, borders */ --muted: #8c8c88; /* Secondary text, inactive icons */ --ink: #1a1a18; /* Primary text, active states, buttons */ --accent: #1a1a18; /* Alias for --ink, always matches */ --line: #e2e2de; /* Borders, dividers, timeline tracks */ /* Spatial System */ --sidebar: 260px; /* Fixed sidebar width */ --hdr: 72px; /* Header height */ --radius: 14px; /* Default card border radius */ /* Effects */ --shadow: 0 2px 24px rgba(0,0,0,.06); /* Standard drop shadow */ } ``` ### 1.2 Typography System **Dual-font architecture:** 1. **Cormorant Garamond** (Serif, weights: 300, 400, 600) - Exclusive use: Branding, logos, page titles, large statistics - Conveys premium, academic feel - Applied to: `.login-logo`, `.sidebar-brand-name`, `.page-title` 2. **DM Sans** (Sans-serif, weights: 300, 400, 500) - Primary workhorse for all UI elements - Default `body` font family - Applied to: Buttons, inputs, body text, navigation **Font weights usage:** - 300 (Light): Elegant headings, taglines - 400 (Regular): Body text, UI labels - 500 (Medium): Button text, section headers, emphasis - 600 (Semibold): Card titles, badges ### 1.3 Color Usage Patterns | Token | Use Case | Examples | |-------|----------|----------| | `--white` | Card bgs, sidebar bg, hover targets | `.login-card`, `.sidebar`, `.modal` | | `--off` | App background, subtle hover | `body bg`, `.nav-item:hover` | | `--light` | Lighter hover states | `.tg-btn:hover`, secondary buttons | | `--ink` | Primary text, active states, CTA buttons | Text, `.nav-item.active`, `.btn-primary` | | `--muted` | Disabled text, labels, icons | Form labels, taglines, secondary info | | `--line` | Borders, dividers, strokes | Input borders, card borders, `hr` | | `--mid` | Section labels, de-emphasized text | Nav section labels | --- ## 2. Core Layout Architecture ### 2.1 Two-State Application Model The app has **two mutually exclusive states** controlled by JavaScript: #### State 1: Login Screen (`#login-screen`) - **Position**: Fixed overlay, `z-index: 9999` - **Visibility**: Visible on page load - **Hidden by**: `doLogin()` function (adds `.hide` class) - **Transition**: Opacity + vertical translate (500ms ease) - **Components**: - `.login-card`: Centered authentication card (max-width 420px) - `.login-tabs`: Tabbed interface (ID/Password vs Telegram) - `.form-group`: Input fields with labels - `.btn-primary`: Primary CTA - `.tg-btn`: Secondary Telegram login button #### State 2: Main App (`#app`) - **Display**: `display: none` initially, then `display: flex` when visible - **Layout**: Flex column with fixed sidebar + main content area - **Structure**: Sidebar (fixed) | Main (flex: 1) - **Shown by**: `doLogin()` function (adds `.visible` class) ### 2.2 Sidebar Layout ```html .sidebar (fixed, width: 260px) ├── .sidebar-brand (header section) │ ├── .sidebar-brand-name (Cormorant Garamond) │ └── .sidebar-brand-sub (muted label) │ ├── .sidebar-nav (scrollable, flex: 1) │ ├── .nav-section-label (repeated sections) │ └── .nav-item × N (with icons, interactive) │ └── .sidebar-footer (user profile section) └── .user-chip (avatar + name) ``` **Sidebar Behavior:** - Fixed width at 260px (desktop) - On mobile (<768px): Translates off-screen by default, toggles with hamburger - Active nav item: `.nav-item.active` (ink bg, white text) - Navigation events: `setNav(el)` - removes active from all, adds to clicked item, closes sidebar ### 2.3 Main Content Area ``` #main ├── header (.header) │ ├── .header-left │ │ ├── .hamburger (mobile only) │ │ └── .breadcrumb │ └── .header-right │ ├── .icon-btn (notifications) │ └── .icon-btn (settings) │ └── .content (scrollable, padding: 40px) ├── .page-title (Cormorant Garamond) ├── .page-sub (muted text) ├── .stats-row (3-col CSS grid) ├── .section-title × N ├── .timeline (vertical stepper) └── .submit-card (form component) ``` --- ## 3. Interactive Components ### 5.1 Buttons **Primary Button (`.btn-primary`)** - Background: `--ink` (dark) - Color: `--white` - Padding: 14px (height ≈ 44px) - Border-radius: 10px - Width: 100% (fills container) - Hover: `opacity: .85; transform: translateY(-1px);` - Active: `transform: translateY(0);` **Ghost Button (`.btn-ghost`)** - Background: `--off` or transparent - Color: `--ink` - Border: 1px solid `--line` (optional) - Padding: 10px 16px - Hover: `background: --light;` **Send Button (`.btn-send`)** - Similar to primary, but with flex layout for icon - Gap: 6px between text and SVG ### 5.2 Icon Buttons (`.icon-btn`) - Size: 40px × 40px - Border: none, background: transparent - SVG: 18px × 18px, stroke-width: 1.8 - Color: `--muted` by default, `--ink` on hover - Transition: color 200ms, background 150ms ### 5.3 Input Fields **Text & URL Inputs (`.form-group input`, `.link-input`)** - Padding: 13px 16px - Border: 1px solid `--line` - Border-radius: 10px - Background: `--off` (unfocused), `--white` (focused) - Font: DM Sans, 14px - Focus: `border-color: --ink;` - Transition: 200ms on border & background **Textarea (`.submit-textarea`)** - Same base styling as inputs - Min-height: 100px - Resize: vertical only (recommended) --- ## 6. Mobile Responsiveness ### 6.1 Breakpoint: 768px (tablet/mobile) **Sidebar Changes:** ```css @media (max-width: 768px) { .sidebar { transform: translateX(-100%); /* Hidden by default */ transition: transform 300ms ease; } .sidebar.open { transform: translateX(0); z-index: 1000; /* Appears above overlay */ } #overlay { display: none; /* Hidden by default */ position: fixed; inset: 0; background: rgba(0,0,0,.2); z-index: 500; } #overlay.visible { display: block; } } ``` **Main Content Adjustments:** - Remove fixed sidebar width constraint - Padding: 20px (reduced from 40px) - Grid layouts: Collapse to single column **Navigation:** - Hamburger icon (`.hamburger`) visible - Click toggles sidebar and overlay - Clicking nav item closes sidebar automatically ### 6.2 Responsive Grid Patterns **Stats Row:** ```css .stats-row { display: grid; grid-template-columns: repeat(3, 1fr); /* Desktop: 3 cols */ gap: 16px; } @media (max-width: 768px) { .stats-row { grid-template-columns: 1fr; /* Mobile: 1 col */ } } ``` **Timeline:** - Remains vertical on all breakpoints - Card padding reduced on mobile (16px instead of 24px) --- ## 7. Vanilla JavaScript API ### 7.1 Core Functions **`switchTab(id)`** - **Purpose**: Toggle between login tabs ("id" or "tg") - **Behavior**: - Adds/removes `.active` class on tab buttons - Shows/hides corresponding `.tab-panel` (by ID) - **Example**: `switchTab('id')` → activates ID/Password tab - **DOM Targets**: `.login-tab`, `#panel-id`, `#panel-tg` **`doLogin()`** - **Purpose**: Transition from login screen to main app - **Behavior**: 1. Adds `.hide` class to `#login-screen` (opacity: 0, pointer-events: none) 2. Waits 500ms (matches CSS transition duration) 3. Sets `display: none` on login screen 4. Adds `.visible` class to `#app` (sets display: flex) - **Trigger**: Login button onclick - **No parameters** **`toggleSidebar()`** - **Purpose**: Open/close sidebar on mobile - **Behavior**: - Toggles `.open` class on `#sidebar` - Toggles `.visible` class on `#overlay` - **Trigger**: Hamburger button click **`closeSidebar()`** - **Purpose**: Explicitly close sidebar - **Behavior**: - Removes `.open` from `#sidebar` - Removes `.visible` from `#overlay` - **Trigger**: On nav item click, overlay click **`setNav(el)`** - **Purpose**: Update active navigation state - **Behavior**: 1. Removes `.active` from all `.nav-item` elements 2. Adds `.active` to clicked element 3. Calls `closeSidebar()` - **Parameter**: `el` = DOM element (e.g., `this` from onclick) - **Trigger**: Nav item click **`onFileChange(input)`** - **Purpose**: Display selected files in UI - **Behavior**: - Reads `input.files` (FileList) - Maps filenames to display string with 📎 emoji - Injects into `#file-list` div as `innerHTML` - Clears if no files selected - **Parameter**: `input` = `` element - **Trigger**: File input `onchange` event - **Example**: `onchange="onFileChange(this)"` **`showToast()`** - **Purpose**: Show temporary success notification - **Behavior**: 1. Adds `.show` class to `#toast` element 2. Waits 3000ms (3 seconds) 3. Removes `.show` class (fades out) - **No parameters** - **Trigger**: "Submit" button click - **Customization**: Modify toast text in HTML (`#toast` content) ### 7.2 Event Binding Patterns All event handlers are inline in HTML (no event listeners in JS): ```html

Strict Development Rules for AI 🚨 ### 8.1 Class Naming Conventions - **Use semantic, flat names**: `tl-card`, `btn-primary`, `icon-btn`, `upload-zone` - **Avoid deep nesting**: Never use BEM-style multi-level (e.g., avoid `tl-card__header__title`) - **Prefix patterns**: - `btn-*` for buttons - `icon-*` for icon elements - `tl-*` for timeline components - `stat-*` for statistics - `nav-*` for navigation ### 8.2 No External Dependencies **Allowed:** - Google Fonts (Cormorant Garamond, DM Sans) - Inline SVGs (stroke-based icons) - Native HTML5 features (file input, form elements) - Vanilla JavaScript (ES6 OK, no transpilation needed) **Prohibited:** - React, Vue, Angular, Svelte, or any framework - Tailwind CSS, Bootstrap, or CSS frameworks - FontAwesome or external icon libraries - jQuery or other DOM libraries - Build tools or bundlers ### 8.3 Inline SVG Icon Standards **Always use `stroke="currentColor"` or explicit color variables:** ```html ``` **Icon size conventions:** - Header/nav icons: 16px – 18px - Large interactive icons: 24px – 32px - Small badge icons: 12px – 14px - Set parent container size; SVG scales via `viewBox` ### 8.4 Responsiveness Checklist For every new component, ensure: - [ ] Padding/margin adjusts for mobile (<768px) - [ ] Grid layouts collapse to 1 column - [ ] Text size is readable on mobile (min 13px body) - [ ] Touch targets are ≥44px (for buttons/icons) - [ ] Flexbox/grid wraps correctly with `flex-wrap` or `grid-auto-flow` - [ ] Sidebar integration works (doesn't break on toggle) - [ ] No horizontal scroll except for explicitly scrollable sections ### 8.5 CSS Transition Standards - Button hover: `opacity .2s, transform .15s` - Color changes: `.2s` ease (smooth) - Sidebar/overlay: `.3s` ease (structural changes) - Navbar/header: `.2s` ease (appearance) - Toast/notifications: `.5s` ease (entry/exit) - Default easing: `ease` or `ease-out` for natural feel ### 8.6 Accessibility Basics While not a full WCAG audit, include: - `aria-label` on icon buttons: `
